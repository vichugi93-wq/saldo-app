Crea una aplicación web completa de finanzas personales llamada "Saldo" con autenticación, sistema freemium con cobro manual, análisis IA, metas de ahorro, soporte multi-moneda y base de datos en Supabase.

## Nombre y marca
- Nombre: Saldo
- Tagline: "Tus finanzas, bajo control"
- Colores: fondo #0f0f0f, verde #22c55e (ingresos/positivo), rojo #ef4444 (gastos/negativo), amarillo #eab308 (metas/advertencia/pendiente), gris #71717a (textos secundarios)
- Tipografía: Inter o System UI

## Stack tecnológico
- Frontend: React + Vite + TypeScript
- Estilos: Tailwind CSS
- Gráficos: Recharts
- IA: Anthropic API (claude-sonnet-4-6)
- Autenticación y base de datos: Supabase
- Storage de comprobantes: Supabase Storage
- ORM: Supabase JS Client (@supabase/supabase-js, usando el nuevo sistema de claves sb_publishable_ / sb_secret_)

## Configuración de Supabase

### Tablas necesarias (incluir SQL completo para crearlas)
```sql
create table profiles (
  id uuid references auth.users primary key,
  name text,
  currency text default 'PYG',
  plan text default 'basic',
  plan_expires_at timestamptz,
  is_admin boolean default false,
  created_at timestamptz default now()
);

create table transactions (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  type text not null,
  amount numeric not null,
  category text not null,
  description text,
  date date not null,
  created_at timestamptz default now()
);

create table goals (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  name text not null,
  target_amount numeric not null,
  current_amount numeric default 0,
  deadline date,
  category text,
  status text default 'active',
  created_at timestamptz default now()
);

create table ai_analyses (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  content text not null,
  created_at timestamptz default now()
);

create table family_profiles (
  id uuid primary key default gen_random_uuid(),
  owner_id uuid references auth.users not null,
  name text not null,
  color text,
  currency text default 'PYG',
  created_at timestamptz default now()
);

create table payment_requests (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  plan_requested text not null,
  receipt_url text not null,
  reference_note text,
  status text default 'pending',
  admin_note text,
  reviewed_by uuid references auth.users,
  reviewed_at timestamptz,
  created_at timestamptz default now()
);
```

### Row Level Security (RLS)
- Habilitar RLS en todas las tablas
- Cada usuario solo lee y escribe sus propios datos
- En payment_requests: el usuario crea las suyas, solo el admin (is_admin = true) actualiza el status
- Incluir las políticas SQL necesarias

### Supabase Storage
- Bucket `comprobantes` (privado)
- Usuario sube solo a su propia carpeta ({user_id}/...)
- Admin puede leer todos los archivos
- Formatos: jpg, png, pdf — máximo 5MB

### Autenticación Supabase
- Email + contraseña
- Recuperación de contraseña por email
- Confirmación de email al registrarse

## Sistema multi-moneda

### Monedas disponibles
₲ Guaraní Paraguayo (PYG, por defecto), $ Peso Argentino (ARS), $ Dólar Estadounidense (USD), R$ Real Brasileño (BRL), $ Peso Uruguayo (UYU), $ Peso Chileno (CLP), $ Peso Colombiano (COP), $ Peso Mexicano (MXN), S/. Sol Peruano (PEN), Bs. Boliviano (BOB), € Euro (EUR)

### Comportamiento
- Moneda guardada en profiles
- Símbolo dinámico en formularios, tarjetas, gráficos, exportaciones
- IA menciona la moneda del usuario en su análisis
- Cada perfil familiar tiene su propia moneda
- Formato por moneda:
  * PYG: sin decimales (₲ 150.000)
  * USD, EUR: 2 decimales ($ 150.00)
  * ARS, CLP, COP: sin decimales ($ 150.000)
  * BRL: 2 decimales con coma (R$ 150,00)
  * Resto: 2 decimales

## Sistema de planes Freemium con cobro manual

### Plan Básico — Gratis
- Hasta 50 transacciones por mes
- 1 meta de ahorro activa
- Dashboard con gráficos básicos
- Sin análisis IA, sin exportar reportes
- Marca de agua "Saldo Básico" en footer
- Periodo de gracia: todo usuario nuevo recibe 30 días de plan Pro gratis al registrarse

### Plan Pro — $4 USD/mes (o equivalente en moneda local)
- Transacciones ilimitadas, metas ilimitadas
- Dashboard completo
- Análisis IA ilimitado
- Exportar reportes
- Historial de últimos 5 análisis
- Badge "Pro", sin marca de agua

### Plan Familiar — $8 USD/mes (o equivalente en moneda local)
- Todo lo de Pro
- Hasta 4 perfiles familiares
- Dashboard familiar consolidado
- Comparativa de gastos entre miembros
- Meta compartida
- Badge "Familia"

### Flujo de cobro manual
1. Al vencer el periodo de prueba (o si Básico quiere upgrade), mostrar pantalla de planes con instrucciones de transferencia bancaria
2. Pantalla muestra datos de cuenta (placeholder editable), monto según plan y moneda
3. Usuario sube foto/PDF del comprobante
4. Usuario opcionalmente agrega nota de referencia
5. Se crea registro en payment_requests con status 'pending'
6. Usuario ve: "Tu pago está en revisión. Te avisaremos cuando se active tu plan (normalmente en menos de 24hs)"
7. El plan permanece en su estado actual hasta aprobación

### Validación del plan
- Plan y vencimiento guardados en profiles
- Al iniciar sesión, verificar vigencia
- Si venció, degradar automáticamente a Básico y notificar
- Banner si el plan vence en menos de 5 días

## Panel de administración (solo is_admin = true)

### Acceso
- Ruta /admin solo visible si profiles.is_admin = true
- Si no-admin intenta acceder, redirigir al dashboard

### Funcionalidades
- Lista de payment_requests, pendientes primero
- Por solicitud: nombre, email, plan solicitado, monto esperado, fecha, nota de referencia
- Botón para ver/descargar comprobante
- Botones "Aprobar" y "Rechazar"
- Al aprobar: actualizar profiles.plan, plan_expires_at (+30 días desde aprobación), status='approved', guardar reviewed_by y reviewed_at
- Al rechazar: pedir admin_note obligatorio, status='rejected'
- Filtros: pendientes / aprobadas / rechazadas / todas
- Estadísticas: total usuarios, usuarios por plan, solicitudes pendientes
